# MessengerWalkieTalkie

Клиент-серверный мессенджер на `Qt 6`, `C++17` и `CMake`.

## Что реализовано

- бинарный протокол `[4 bytes size][JSON payload][4 bytes CRC32]`
- защита от TCP fragmentation и проверка целостности пакетов
- регистрация и логин пользователей
- список пользователей онлайн
- публичные и приватные сообщения
- `seq` + `ack` + retry с exponential backoff `2s -> 4s -> 8s -> 16s -> 32s`
- heartbeat `ping/pong` и разрыв при таймауте `15s`
- многопоточный сервер с распределением клиентов по worker-потокам
- асинхронный логгер с записью в файл и ротацией логов
- клиент `Qt Widgets` с историей сообщений и статусами `sending`, `delivered`, `read`

## Структура

- `Server/` — серверная часть на `Qt Core`, `Qt Network`, `Qt Sql`
- `Client/` — клиентская часть на `Qt Widgets`

## Сборка

Требования:

- `Qt 6`
- `CMake 3.16+`
- компилятор с поддержкой `C++17`

Пример для Windows и Qt 6.8.3:

```powershell
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug -DCMAKE_PREFIX_PATH=C:\Qt\6.8.3\msvc2022_64
cmake --build build
```

## Запуск сервера

```powershell
.\build\Server\server.exe
```

Сервер слушает порт `5555`, хранит пользователей в `users.db`, а логи пишет в директорию `logs/`.

## Запуск клиента

```powershell
.\build\Client\client.exe
```

Шаги:

1. Введите `host` и `port`.
2. Нажмите `Connect`.
3. Выполните `Register` или `Login`.
4. Выберите `Broadcast` или конкретного пользователя.
5. Отправляйте сообщения.

## Базовый сценарий проверки

1. Запустите сервер.
2. Откройте два клиента.
3. Зарегистрируйте двух пользователей.
4. Выполните логин с обеих сторон.
5. Проверьте обновление списка онлайн.
6. Проверьте публичные и приватные сообщения.
7. Отключите один клиент и убедитесь, что список пользователей обновился.
