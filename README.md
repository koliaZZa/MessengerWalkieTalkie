# MessengerWalkieTalkie

MessengerWalkieTalkie - клиент-серверный мессенджер на `Qt 6`, `C++17` и `CMake`.

Проект состоит из двух приложений:
- `Server` - TCP-сервер на `Qt Core`, `Qt Network`, `Qt Sql`
- `Client` - GUI-клиент на `Qt Widgets`, `Qt Network`, `Qt Sql`

## Возможности

- собственный бинарный протокол: `[4 bytes size][JSON payload][4 bytes CRC32]`
- защита от TCP fragmentation и проверка целостности пакетов через `CRC32`
- регистрация и логин пользователей
- автоматический вход сразу после успешной регистрации
- список пользователей онлайн
- публичные и приватные сообщения
- подтверждение доставки через `seq`, `ack` и retry с exponential backoff
- статусы сообщений `sending`, `delivered`, `read`
- heartbeat `ping/pong` и автоматическое переподключение клиента
- хранение истории сообщений на сервере в `SQLite`
- хранение локальной истории и последней сессии на клиенте
- офлайн-режим клиента для просмотра последней истории в режиме `read-only`, если сервер недоступен
- отложенная доставка приватных сообщений: если получатель офлайн, сообщение сохраняется и доставляется после его логина
- сохранение истории общего чата `Broadcast` на сервере
- отдельные списки `Dialogs` и `Online users` в окне чата
- создание нового диалога через выбор онлайн-пользователя или ввод существующего `username`
- метки времени у сообщений в интерфейсе
- автоматический деплой Qt DLL через `windeployqt` после сборки

## Архитектура

```text
Client (Qt Widgets)
        |
        | TCP
        v
Server (Qt Core + QTcpServer + workers)
```

### Сервер

- `QTcpServer` принимает подключения
- соединения распределяются по worker-потокам
- каждое подключение обслуживается объектом `Connection`
- авторизация и история сообщений хранятся в `SQLite`
- логирование вынесено в отдельный поток

### Клиент

- стартует с окна подключения
- показывает статус подключения: `Connecting`, `Connected`, `Disconnected`, `Reconnecting`
- позволяет менять `IP` и `port` через окно настроек подключения
- если сервер недоступен, но есть последняя сессия, можно открыть офлайн-режим
- в офлайн-режиме клиент не отправляет сообщения, но продолжает переподключаться в фоне
- после успешного подключения открывает окно логина
- если предыдущая сессия восстановима, выполняет автологин и сразу открывает окно мессенджера
- смена сервера доступна из окна подключения, окна логина и окна чата

## Завершение работы

- клиент при закрытии явно закрывает TCP-соединение, останавливает таймеры ping, retry и reconnect и освобождает сетевые ресурсы
- сервер при завершении закрывает активные клиентские соединения, корректно останавливает worker-потоки и завершает логгер
## История сообщений

### На сервере

- история хранится в `SQLite`
- приватные сообщения сохраняются даже для офлайн-получателей
- после логина получателя сервер доставляет накопленные private-сообщения
- сообщения `Broadcast` тоже сохраняются и доступны в истории общего чата

### На клиенте

- локальная история хранится в `SQLite`
- сохраняется последняя успешная сессия
- при недоступном сервере можно открыть историю в офлайн-режиме
- при `Logout` локальная история текущего пользователя очищается
- окно чата показывает время отправки каждого сообщения

## Требования

- `Qt 6`
- `CMake 3.16+`
- компилятор с поддержкой `C++17`
- `Ninja`
- Visual Studio 2022 Build Tools или полная Visual Studio 2022 для `MSVC`

## Запуск сервера

Запуск на порту по умолчанию `5555`:

```powershell
...\Server\server.exe
```

Запуск с явным портом:

```powershell
...\Server\server.exe 50000
```

или

```powershell
...\Server\server.exe --port 50000
```

Сервер слушает `AnyIPv4`, то есть принимает подключения на всех IPv4-интерфейсах машины.

## Запуск клиента

```powershell
.\build\Desktop_Qt_6_8_3_MSVC2022_64bit-Debug\Client\client.exe
```

После запуска клиент:
- открывает окно подключения
- пытается подключиться к серверу
- позволяет сменить `IP` и `port`
- при успешном соединении открывает окно логина или сразу переходит в окно мессенджера, если сессия уже восстановлена
- при недоступном сервере может открыть последнюю локальную историю в офлайн-режиме

## Пример использования

1. Соберите `server` и `client`.
2. Запустите сервер:
   `server.exe --port 5555`
3. Запустите клиент.
4. В окне подключения укажите `127.0.0.1` и `5555`.
5. Зарегистрируйте нового пользователя или войдите в существующий аккаунт.
6. После логина откроется окно `Walkie Talkie`, где можно:
   - отправлять `Broadcast`-сообщения
   - создавать приватные диалоги
   - выбирать пользователя из списка онлайн или ввести существующий `username`
   - просматривать историю сообщений
   - видеть время отправки и статус доставки сообщений

## Примечания

- Для подключения по публичному IP обычно нужно открыть или пробросить `TCP`-порт сервера на роутере.
- Если сервер недоступен, клиент может продолжать показывать локальную историю в офлайн-режиме.
- Для релизной раздачи рекомендуется использовать `Release`-сборку.
